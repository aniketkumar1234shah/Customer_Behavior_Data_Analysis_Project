# Customer-Behavior-Data-Analysis

# End-to-End Data Analytics Project | Python • PostgreSQL • Power BI

## 📌 Project Overview

Customer Behavior Analysis is an end-to-end data analytics project designed to simulate a real-world retail company use case.

The project demonstrates how raw customer transaction data can be transformed into meaningful business insights using Python, PostgreSQL, and Power BI.

It follows a complete analytics workflow — from data loading and cleaning to business analysis and dashboard development — similar to what data analysts perform in professional environments.

## 🎯 Business Objective

A retail company wants to analyze customer purchasing patterns to:

- Identify high-value customers

- Understand buying trends and seasonality

- Evaluate product category performance

- Analyze revenue distribution

- Support data-driven decision-making

- The goal is to convert transactional data into actionable insights for business growth.

## 🛠 Tech Stack

- Python (Pandas, NumPy) – Data Cleaning & EDA

- PostgreSQL – Business Querying & Data Analysis

- Power BI – Interactive Dashboard & KPI Visualization

- GitHub – Version Control & Project Documentation

 ## 🔹 Project Workflow
### 1️⃣ Data Loading in Python

The dataset was imported into Python for initial exploration and validation.

Key activities:

- Inspecting dataset structure

- Understanding data types

- Identifying missing values

- Performing preliminary data checks

- This step ensures data quality before analysis.

### 2️⃣ Data Cleaning & Preparation

Data cleaning was performed to make the dataset analysis-ready.

Activities included:

- Handling missing values

- Removing duplicate records

- Standardizing column formats

- Converting date fields

This stage ensures accuracy and reliability in downstream analysis.

### 3️⃣ Exploratory Data Analysis (EDA)

EDA was conducted to uncover patterns and trends in customer purchasing behavior.

Key analysis areas:

- Sales distribution analysis

- Revenue by product category

- Monthly and seasonal sales trends

- ustomer purchase frequency

- Revenue contribution by customer segment


### 4️⃣ SQL Analysis in PostgreSQL

The cleaned dataset was imported into PostgreSQL to perform structured business analysis.

Analysis included:

- Customer revenue ranking

- Category-wise performance analysis

- Monthly revenue trends

- Identifying repeat vs new customers

- Revenue aggregation for business KPIs

This step simulates real-world database querying environments used in companies.

### 5️⃣ Power BI Dashboard

An interactive Power BI dashboard was developed to present insights in a business-friendly format.

Dashboard Features:

- KPI Cards (Total Revenue, Total Customers, Average Order Value)

- Monthly Sales Trend Visualization

- Category-wise Revenue Breakdown

- Top Customers Analysis

- Interactive Filters for dynamic exploration

The dashboard enables stakeholders to monitor performance and make strategic decisions.
